# Отчеты о дефектах (Bug Reports - API)

---

### BUG-API-001: Некорректный статус-код при отправке пустого POST-запроса
- **Приоритет:** Средний (Medium)
- **Метод:** POST
- **URL:** `https://jsonplaceholder.typicode.com/posts`
- **Описание:** При отправке запроса с пустым телом `{}` сервер создает пустой ресурс.
- **Ожидаемый результат:** Статус-код `400 Bad Request` или ошибка валидации (поля title/body обязательны).
- **Фактический результат:** Статус-код `201 Created`. Создан пустой объект с `id: 101`.

---

### BUG-API-002: Сервер принимает строку вместо числа в поле userId (POST)
- **Приоритет:** Низкий (Low)
- **Метод:** POST
- **Payload:** `{"title": "Test", "body": "Test", "userId": "ABC"}`
- **Ожидаемый результат:** Статус-код `400 Bad Request`. Ошибка типа данных (userId должен быть integer).
- **Фактический результат:** Статус-код `201 Created`. Сервер сохранил строковое значение "ABC" в числовое поле.

---

### BUG-API-003: Отсутствие заголовка Content-Type: application/json в ответе
- **Приоритет:** Средний (Medium)
- **Описание:** Заголовок ответа сервера не указывает на формат данных JSON.
- **Ожидаемый результат:** В заголовках (Headers) ответа присутствует `Content-Type: application/json; charset=utf-8`.
- **Фактический результат:** Заголовок отсутствует или содержит `text/html`, что затрудняет автоматический парсинг ответа клиентом.
