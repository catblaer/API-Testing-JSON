# Тест-кейсы для тестирования API (JSONPlaceholder)

В данном документе описаны сценарии проверки работоспособности REST API для модуля "Посты" (Posts).

---

## Перечень тест-кейсов

### TC-API-001: Получение списка всех постов
•   **Метод:** GET
•   **Эндпоинт (URL):** `https://jsonplaceholder.typicode.com/posts`
•   **Описание:** Проверка возможности получения всех доступных постов из базы данных.
•   **Ожидаемый результат:** 
    1. Статус-код `200 OK`.
    2. Тело ответа — массив объектов JSON.
    3. В массиве ровно 100 объектов.
•   **Фактический результат:** Статус-код `200 OK`, получен JSON-массив из 100 элементов. Структура объектов корректна (userId, id, title, body).
•   **Статус:** Пройден (Passed)

---

### TC-API-002: Получение данных конкретного поста (по ID)
•   **Метод:** GET
•   **Эндпоинт (URL):** `https://jsonplaceholder.typicode.com/posts/1`
•   **Описание:** Проверка получения данных поста с существующим ID = 1.
•   **Ожидаемый результат:** 
    1. Статус-код `200 OK`.
    2. В ответе объект с `id: 1`.
•   **Фактический результат:** Статус-код `200 OK`, данные поста #1 успешно загружены.
•   **Статус:** Пройден (Passed)

---

### TC-API-003: Запрос несуществующего поста (Негативный сценарий)
•   **Метод:** GET
•   **Эндпоинт (URL):** `https://jsonplaceholder.typicode.com/posts/9999`
•   **Описание:** Проверка реакции сервера на запрос ID, которого нет в базе.
•   **Ожидаемый результат:** 
    1. Статус-код `404 Not Found`.
    2. Тело ответа пустое `{}`.
•   **Фактический результат:** Статус-код `404 Not Found`, данные не найдены.
•   **Статус:** Пройден (Passed)

---

### TC-API-004: Создание нового поста (POST)
•   **Метод:** POST
•   **Эндпоинт (URL):** `https://jsonplaceholder.typicode.com/posts`
•   **Заголовки (Headers):** `Content-Type: application/json`
•   **Тело запроса (Body):**
    json
    {
      "title": "QA Portfolio Test",
      "body": "This is a test post for API documentation",
      "userId": 1
    }
    ```
•   Ожидаемый результат: 
    1. Статус-код 201 Created.
    2. В ответе возвращаются отправленные данные + новый id: 101.
•   Фактический результат: Статус-код 201 Created, объект успешно создан, присвоен ID 101.
•   Статус: Пройден (Passed)

---

 TC-API-005: Обновление существующего поста (PUT)
•   Метод: PUT
•   Эндпоинт (URL): https://jsonplaceholder.typicode.com/posts/1
•   Тело запроса (Body):
    
    {
      "id": 1,
      "title": "Updated Title",
      "body": "Updated Body Text",
      "userId": 1
    }
    
•   Ожидаемый результат: 
    1. Статус-код 200 OK.
    2. Данные в ответе соответствуют измененным значениям.
•   Фактический результат: Статус-код 200 OK, заголовок и тело поста успешно обновлены.
•   Статус: Пройден (Passed)

---

 TC-API-006: Удаление поста (DELETE)
•   Метод: DELETE
•   Эндпоинт (URL): https://jsonplaceholder.typicode.com/posts/1
•   Описание: Проверка удаления ресурса по его ID.
•   Ожидаемый результат: 
    1. Статус-код 200 OK (или 204 No Content в зависимости от реализации).
•   Фактический результат: Статус-код 200 OK, ресурс удален (симуляция удаления на JSONPlaceholder).
•   Статус: Пройден (Passed)
